Floating Pins

August 18, 2020 One-minute read
Journal • TIL
chess • iot • arduino

IoT

“Reading input from mechanical devices is easy.
Reading input resiliently from mechanical devices is not."

Problem: Determine the value read by the pin when the mechanical device (such as push button/sensor) is not pressed. How to remove “false-positives” (reading button press when it wasn’t in reality) from the input space?

Causes (potential):

  • Mechanical fault in the device.
  • Problem with the circuit.

Concept: The idea of floating pins - pin with an indeterminant/fluctuating value.

Solution: Use a “pull up” resistor to tie the floating pin to a specific state.

  • Default state of the input pin is set to HIGH.
  • On activation (button press), the input read is LOW.

Implementation:

  1. Connect a resistor to 5V (HIGH) to the push button.
  2. Set the pin mode to INPUT_PULLUP instead of INPUT; taking advantage of the hardware features. 🤷‍♂️
